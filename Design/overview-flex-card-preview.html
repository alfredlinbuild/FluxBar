Overview 可切换状态卡方案

这块区域可以做成“可替换状态卡位”，但不建议做完全开放的自由内容区。更合理的方向是提供有限模式， 保持同一张卡的结构一致：顶部小标题、中间主状态、底部脚注说明。

哪个更精确？ 这种界面预览里，HTML + CSS 更精确。因为它是基于真实布局流的渲染， 会处理字号、换行、间距、对齐和响应式约束；单纯“HTML 网页”其实离不开 CSS，真正精确的是两者一起。 比起手工画 SVG 或概念图，更接近后面落到 SwiftUI 的真实结果。

建议的模式

  1. Thermal：默认模式，最符合 FluxBar 定位。
  2. Weather：最适合第一批扩展，既有趣也有用。
  3. Focus：偏氛围型，适合工作状态展示。
  4. Mood：可以做，但更像个性化装饰，建议排后。

设计原则：不管切换成什么内容，这张卡都应保持同一张卡片系统里的成员，而不是变成一个风格跳脱的小挂件。

方案定义

把现在的 Thermal State 改成 Spotlight Card。 用户可以在设置里切换卡片模式，但每种模式沿用同一套结构和高度，避免破坏 Overview 的平衡。

Thermal Weather Focus Mood

Thermal

保留系统监控主定位，最稳妥，也是默认值。

Weather

最值得先做的一种有趣扩展，属于“环境信息”，不违和。

Focus

展示当前工作状态或一句短状态词，偏 productivity。

Mood

更个性化，但离系统工具更远，建议做成后续可选项。

效果预览

左侧网络卡保持不变，右侧用相同结构展示不同模式。这样最容易判断“有趣”是否会破坏整体产品感。

结论

如果你想让这块区域“更有趣但不跑偏”，最适合的第一版是： Thermal / Weather 两种模式。它们都属于“状态信息”，和 FluxBar 的工具属性兼容度最高。 Focus 可以作为第二阶段；Mood 更适合放到更个性化的版本里。
